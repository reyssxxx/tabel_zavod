# Кейс №6 — Табель учета рабочего времени

## Заказчик
АО «ТАНТК им. Г.М. Бериева»  
Количество сотрудников: 3000+

## Описание задачи

На предприятии ведётся учет рабочего времени сотрудников (инженеры, сборщики, операторы ЧПУ, отдел закупок и др.).  
Необходимо разработать приложение (веб или десктоп) для цифрового учета рабочего времени сотрудников.

Система должна помочь отделу кадров и бухгалтерии:

- фиксировать ежедневные отметки сотрудников
- формировать табель за месяц
- получать отчеты по подразделениям
- выгружать данные

---

# Функциональные требования

## 1. Справочник сотрудников

### Возможности:
- Добавление сотрудника
- Редактирование сотрудника
- Удаление сотрудника
- Пометка сотрудника как неактивного (уволен)

### Поля сотрудника:
- ФИО
- Должность
- Цех / отдел
- Табельный номер
- Статус (активен / неактивен)

Неактивные сотрудники не отображаются в основных списках, но их данные сохраняются.

---

## 2. Учёт рабочего времени

Ежедневная отметка сотрудника.

### Возможные статусы:
- `Я` — Явка (полный рабочий день)
- `ОТ` — Отпуск
- `Б` — Больничный
- `К` — Командировка
- `П` — Прогул
- `С` — Сокращённый день

Каждая отметка:
- Привязана к конкретному сотруднику
- Привязана к конкретной дате

---

## 3. Просмотр табеля за период

### Возможности:
- Выбор месяца и года
- Отображение таблицы:

| Сотрудник | 1 | 2 | 3 | ... | 31 |
|------------|---|---|---|-----|----|
| Иванов И.И | Я | Я | Б | ... | Я |

- Итог по каждому сотруднику:
  - Количество отработанных дней
  - Количество часов (если реализовано)

---

## 4. Отчёты

### Отчет по подразделению за месяц:
- Количество сотрудников
- Сколько работали
- Сколько были в отпуске
- Сколько на больничном

### Экспорт:
- Выгрузка отчета в:
  - TXT
  - CSV

---

## 5. Дополнительные требования

### 1. Автоматический подсчет часов
- Рабочий день = 8 часов (или настраиваемый параметр)
- Автоматический расчет итоговых часов за месяц

### 2. Праздничные и выходные дни
- Подсветка выходных
- Подсветка праздничных дат

### 3. Роли пользователей
- Мастер / Начальник цеха — редактирование только своего подразделения
- Бухгалтер — просмотр без редактирования
- Администратор — полный доступ

### 4. Поиск и фильтрация
- По фамилии
- По табельному номеру
- По подразделению

### 5. Импорт / экспорт
- Импорт сотрудников из CSV
- Экспорт табеля в Excel

# Нефункциональные требования

## База данных

### Обязательные таблицы:

### 1. Employees (Сотрудники)
- id
- full_name
- position
- department
- personnel_number
- is_active

### 2. MarkTypes (Типы отметок)
- id
- code (Я, ОТ, Б, К, П, С)
- name

### 3. TimeRecords (Учёт времени)
- id
- employee_id (FK)
- date
- mark_type_id (FK)

### СУБД:
- SQLite / PostgreSQL / MySQL
