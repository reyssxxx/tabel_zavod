1. НАЗНАЧЕНИЕ ПРИЛОЖЕНИЯ
═══════════════════════════════════════════════════════════════════════════════

Веб-приложение для учёта рабочего времени в крупном производственном предприятии
(АО "ТАНТК им. Г.М. Бериева", 3000+ сотрудников). Система позволяет:

  • Администраторам управлять сотрудниками и доступом
  • Руководителям цехов отслеживать табели своих подразделений
  • Бухгалтерам просматривать все данные, но без прав редактирования
  • Вводить ежедневные отметки (Я - явка, ОТ - отпуск, Б - болезнь, К - командировка,
    П - простой, С - сокращённый день)
  • Генерировать отчеты по отделам и экспортировать данные (Excel, CSV, TXT)
  • Настраивать праздничные дни, графики работы, должности и зарплаты

Фокус на функциональность, а не на дизайн (проект создан под хакатон).


2. ФРОНТЕНД (Front-end)
═══════════════════════════════════════════════════════════════════════════════

Технологии:
  • Next.js 16 с App Router (новая архитектура маршрутизации)
  • React × TypeScript для типобезопасности и разработки
  • Tailwind CSS v4 для быстрого стилизирования
  • shadcn/ui — библиотека готовых компонентов на базе Tailwind

Архитектура:
  • Точка входа: src/app/layout.tsx (корневой layout с SessionProvider)
  • Публичная страница: src/app/auth/login/page.tsx (вход без сайдбара)
  • Защищенные страницы: src/app/(dashboard)/* с логиновой группой маршрутов
  • Макет: src/app/(dashboard)/layout.tsx с сайдбаром и заголовком
  • Основные страницы:
    - /dashboard — главная
    - /employees — управление сотрудниками (CRUD, поиск, CSV-импорт)
    - /timesheet — табель рабочих часов (месячная сетка, встроенное редактирование)
    - /reports — отчеты по отделам
    - /settings — настройки (праздники, должности, графики, работные часы)
    - /audit — логирование всех изменений

Компоненты:
  • Переиспользуемые UI-элементы в src/components/ui/
  • Прикладные компоненты (app-header, app-sidebar, timesheet-grid, report-card и т.д.)
  • Диалоги для добавления/редактирования (employee-form, csv-import-dialog)
  • Встроенное редактирование ячеек (timesheet-cell, cell-popup)

Клиентская логика хранится в компонентах, серверная — в серверных функциях (src/lib/)
и API-маршрутах.

Интерфейс полностью на русском языке.


3. БЭКЕНД (Back-end)
═══════════════════════════════════════════════════════════════════════════════

Архитектура:
  • Next.js API Routes (src/app/api/) вместо отдельного бэк-сервера
  • RESTful API для управления сотрудниками, табелями, отчетами, настройками
  • Серверные функции для критичной логики (избегаем expose логики на клиенте)

Аутентификация и авторизация:
  • NextAuth v5 (beta) с JWT-сессиями
  • Трёхуровневая система ролей:
    - ADMIN: полный доступ ко всем разделам и отделам
    - MANAGER: редактирование только своего отдела (фильтруется по departmentId)
    - ACCOUNTANT: чтение всех данных, редактирование запрещено
  • Утилиты в src/lib/auth-utils.ts:
    - getCurrentUser() — получить текущего пользователя из сессии
    - requireRole() — проверить роль в API-маршрутах
    - canEditDepartment() — проверить доступ к редактированию отдела
  • На щите приложения есть переключатель ролей для демо (позволяет переаутентифицироваться)

API-маршруты:
  • /api/employees — CRUD операции с сотрудниками
  • /api/employees/[id] — редактирование одного сотрудника
  • /api/employees/import — CSV импорт
  • /api/timerecords — создание/обновление табельных отметок
  • /api/reports — генерация отчетов
  • /api/departments, /api/holidays, /api/mark-types, /api/settings/* — управление справочниками
  • /api/audit — логирование действий
  • /api/export/* — экспорт в Excel/CSV/TXT
  • /api/auth/[...nextauth] — обработчик NextAuth

Утилиты в src/lib/:
  • auth.ts — конфигурация NextAuth
  • auth-utils.ts — функции проверки доступа и получения сессии
  • db.ts — singleton Prisma-клиента для работы с БД
  • constants.ts — константы (месяцы на русском, ярлыки ролей, работные часы в день)
  • holidays.ts — кеширование праздников, проверка выходных дней
  • export.ts — экспорт в разные форматы
  • salary.ts — расчеты зарплаты
  • work-hours.ts — расчет рабочих часов
  • audit.ts — логирование изменений

Дата передается через API как 0-индексированный месяц (0 = январь) по
compat JavaScript Date. Даты в базе хранятся как DateTime, но значение имеет только дата.


4. БАЗА ДАННЫХ
═══════════════════════════════════════════════════════════════════════════════

ORM: Prisma 6
СУБД: SQLite (файл prisma/dev.db)
Путь хранения: prisma/dev.db в корне проекта

Основные модели (Prisma schema в prisma/schema.prisma):

  • User — пользователи системы (email, password, role, departmentId)
  • Department — отделы/цеха
  • Employee — сотрудники (имя, должность, дата приёма, активность со статусом активности)
  • TimeRecord — ежедневные отметки за дату (уникальное на [employeeId, date])
  • MarkType — справочник кодов отметок (Я, ОТ, Б, К, П, С)
  • Holiday — календарь праздничных дней
  • Position — должности сотрудников с зарплатой
  • Schedule — графики работы
  • AuditLog — журнал всех изменений (когда, кто, что изменил)
  • AppSettings — глобальные параметры приложения

Миграции:
  • prisma/migrations/ содержит все миграции БД
  • Команда: npx prisma migrate dev --name <имя> — создать и применить миграцию
  • Команда: npx prisma studio — интерактивный редактор БД на localhost:5555
  • Команда: npx prisma db seed — заполнить БД начальными данными из prisma/seed.ts

Начальные данные (seed):
  • 10 сотрудников (в разных отделах)
  • 6 отделов/цехов
  • 3 пользователя с разными ролями (admin, manager, accountant)
  • Праздничные дни (Новый год, праздники РФ и т.д.)

Удаление мягкое: сотрудники не зависят физически, но помечаются isActive = false.

Клиент Prisma: singleton в src/lib/db.ts, инициализируется один раз, переиспользуется везде.

═══════════════════════════════════════════════════════════════════════════════

Next.js 16 (App Router):
  ✓ Полнофункциональный фреймворк (фронт + бэк в одном репозитории)
  ✓ API Routes встроены, не нужно отдельного серверного приложения
  ✓ App Router — новая архитектура со скоростью Turbopack
  ✓ Быстрая разработка для хакатона
  ✓ Хостинг на одной платформе (Vercel, простой VPS, docker)
  ✗ Менее гибкий для очень крупных микросервисных архитектур (но для этого не нужно)

React + TypeScript:
  ✓ React — стандартный UI фреймворк, большое сообщество
  ✓ TypeScript предотвращает ошибки типов на уровне разработки
  ✓ Лучшая IDE-поддержка, автодополнение
  ✓ Проще поддерживать код при масштабировании

Tailwind CSS v4:
  ✓ Utility-first подход: изменения просто через className, без отдельных CSS файлов
  ✓ Быстро стилизировать, не писать много кода
  ✓ Есть тёмная тема из коробки
  ✓ Отлично работает с компонентами

shadcn/ui:
  ✓ Готовые, красивые компоненты (Button, Dialog, Dropdown, Table, etc.)
  ✓ Перекликаются с Tailwind, легко кастомизировать
  ✓ Используют Headless-подход (всё доступно, можно менять)
  ✓ Быстро прототипировать интерфейсы

Prisma 6:
  ✓ Простой ORM с интуитивным языком schema
  ✓ Автоматические миграции (prisma db push, migrate dev)
  ✓ Встроенный Seeding для тестовых данных
  ✓ Prisma Studio — удобный GUI для БД
  ✓ Типобезопасность в querys (автозаполнение в IDE)
  ✓ Нет надобности писать SQL вручную

SQLite:
  ✓ Встроенная БД, файл в проекте, нет необходимости развёртывать сервер PostgreSQL/MySQL
  ✓ Идеально для хакатона, быстрой разработки и демо
  ✓ Легко делать резервные копии (скопировать файл)
  ✓ Поддерживает транзакции, индексы, constraints
  ✓ Легко масштабируется до 100K одновременных соединений (далеко за пределы этого проекта)
  ✗ Не подходит для 10K+ одновременных писателей (но мы не имеем таких требований)

NextAuth v5 (beta):
  ✓ Стандартная библиотека аутентификации для Next.js
  ✓ JWT сессии встроены
  ✓ Легко расширить (добавить custom claims: role, departmentId)
  ✓ Встроенная защита CSRF, secure cookies
  ✓ Поддержка OAuth, email/password, Custom Providers
  ✓ Интеграция с Next.js без лишних слоёв абстракции

Export библиотеки (xlsx, csv-stringify):
  ✓ xlsx (SheetJS) — промышленный стандарт для Excel, поддерживает форматирование
  ✓ csv-stringify — простой CSV экспорт
  ✓ Текстовые отчеты просто через template literals

Модульная архитектура:
  ✓ Разделение на src/components/, src/lib/, src/app/api/ упрощает навигацию и поддержку
  ✓ Каждый модуль (Employees, Timesheet, Reports, Settings) имеет свои страницы, компоненты, API
  ✓ Легко добавить новый функционал отдельно от существующего

7. ОСНОВНЫЕ ВОЗМОЖНОСТИ
═══════════════════════════════════════════════════════════════════════════════

Модуль сотрудников (Employees):
  • Просмотр списка (пагинация, поиск по ФИО)
  • Добавление/редактирование сотрудника (форма с полями: ФИО, должность, дата приёма)
  • Удаление (мягкое удаление — isActive = false)
  • Массовый импорт из CSV
  • Просмотр профиля сотрудника

Модуль табели (Timesheet):
  • Месячная сетка (дни × сотрудники)
  • Встроенное редактирование ячеек (клик на ячейку → выбрать отметку)
  • Цветовое кодирование ячеек (явка зелёная, отпуск синий, болезнь красная)
  • Фильтр по отделу, сотруднику, месяцу
  • Выделение выходных дней на основе праздничного календаря
  • Подсчет фактических рабочих часов

Модуль отчётов (Reports):
  • Отчеты по отделам (статистика отметок, среднее количество дней, итоговые часы)
  • Экспорт в Excel (с форматированием), CSV, TXT
  • Фильтр по отделу, месяцу

Модуль настроек (Settings):
  • Отделы (CRUD)
  • Праздничные дни (добавить, удалить дату)
  • Должности (CRUD с зарплатой)
  • Графики работы (рабочие дни, часы в день)
  • Работные часы в день (по умолчанию 8)

Логирование (Audit):
  • Журнал всех операций (кто, что, когда изменил)
  • Просмотр истории изменений

Ролевой контроль доступа (RBAC):
  • ADMIN: всё редактировать везде
  • MANAGER: только его отдел
  • ACCOUNTANT: только просмотр

Тестовые учётные записи:
  • admin@tabel.ru / admin123 (ADMIN)
  • master@tabel.ru / master123 (MANAGER, "Цех №1")
  • buh@tabel.ru / buh123 (ACCOUNTANT)
  
Демо-переключатель ролей в шапке приложения для тестирования разных уровней доступа.

9. ПРОЦЕСС РАЗРАБОТКИ И ОСОБЕННОСТИ
═══════════════════════════════════════════════════════════════════════════════

Как добавить новый функционал:
  1. Спроектировать в Prisma schema (если нужна новая модель или поля)
  2. Создать миграцию: npx prisma migrate dev --name описание
  3. Добавить API маршруты в src/app/api/что-то/route.ts
  4. Реализовать компоненты в src/components/
  5. Добавить страницу в src/app/(dashboard)/что-то/page.tsx
  6. Применить ролевой контроль (requireRole, canEditDepartment)
  7. Протестировать через Prisma Studio и фронтенд

Особенности:
  • Даты передаются как 0-индексированные месяцы (0 = январь) для совместимости с JS Date
  • В БД хранится DateTime, но значамые только дата (время всегда 00:00:00)
  • Сотрудники удаляются мягко (isActive = false), не стираются из БД
  • Пароли хешируются (не храняются в открытом виде)
  • Все операции логируются в AuditLog для аудита
  • Сессия содержит role и departmentId для быстрой проверки доступа
  • На тёмной теме shadcn/ui компоненты автоматически адаптируются (Tailwind)

Тестирование:
  • Используйте demo пользователей для проверки разных ролей
  • Prisma Studio для прямого просмотра и редактирования БД (localhost:5555)
  • ESLint (pnpm lint) для проверки качества кода
  • Возможна интеграция с Jest/Vitest позже, на хакатоне это невысокий приоритет
