  ЗАЩИТА ПРОЕКТА — ТЕКСТ НА 5 МИНУТ
  «Табель учёта рабочего времени»
  АО «ТАНТК им. Г.М. Бериева»
=========================================================

── ВСТУПЛЕНИЕ (30 сек) ──────────────────────────────────

Мы разработали цифровую систему учёта рабочего времени
для крупного авиастроительного предприятия — ТАНТК им.
Бериева — организации с более чем 3000 сотрудников,
распределённых по нескольким цехам и подразделениям.

Проблема, которую мы решали: на предприятии учёт
ведётся в Excel-таблицах или вручную. Это приводит к
ошибкам, потере данных, невозможности быстро получить
сводную аналитику и рассчитать зарплату.


── СТЕК И АРХИТЕКТУРА (30 сек) ──────────────────────────

Мы выбрали современный production-ready стек:

  • Next.js 16 (App Router) + TypeScript — серверный
    рендеринг, типобезопасность, масштабируемость
  • Tailwind CSS + shadcn/ui — консистентный UI
  • Prisma + SQLite — простое развёртывание без
    отдельного сервера БД (для прода — легко заменить
    на PostgreSQL)
  • NextAuth v5 — JWT-сессии с ролевой моделью

Всё приложение — единый монорепозиторий, запускается
одной командой. Готово к развёртыванию.


── РОЛЕВАЯ МОДЕЛЬ (45 сек) ──────────────────────────────

Это ключевая особенность системы. Мы реализовали
4 роли с разграничением доступа:

  🔴 АДМИНИСТРАТОР — полный доступ ко всем данным,
     настройкам коэффициентов, пользователям

  🔵 МЕНЕДЖЕР ОТДЕЛА (MANAGER) — видит и редактирует
     табель только своего подразделения. Не может
     трогать другие отделы — защита от ошибок

  🟢 БУХГАЛТЕР (ACCOUNTANT) — только чтение,
     выгрузка отчётов и расчёт зарплаты

  🟣 СПЕЦИАЛИСТ ОК (HR) — просмотр данных по
     всем сотрудникам для кадрового учёта

Переключение ролей прямо в интерфейсе — для демонстрации.


── ТАБЕЛЬ (1 мин) ───────────────────────────────────────

Основной рабочий инструмент — сетка табеля.

  • Сразу весь месяц в одном экране — нет перелистывания
  • Цветовое кодирование: зелёный = явка, красный =
    больничный/прогул, синий = отпуск, оранжевый =
    командировка, фиолетовый = сокращённый день
  • Инлайн-редактирование — кликнул на ячейку,
    выбрал отметку, сохранилось без перезагрузки
  • Поддержка сверхурочных часов — отдельное поле
    для каждой записи
  • Учёт совместителей — один человек может числиться
    в двух подразделениях одновременно (реальная
    практика на заводе)
  • Праздничные и сокращённые дни РФ — автоматически
    подсвечиваются по официальному производственному
    календарю 2026 года


── РАСЧЁТ ЗАРПЛАТЫ (1 мин) ──────────────────────────────

Это наш главный технический акцент — автоматический
расчёт на основе реальных норм ТК РФ:

  • Базовая ставка от должности × отработанные часы
  • Сверхурочные по ТК РФ ст. 152:
    первые N часов × коэф. 1.5, остальные × 2.0
  • Выходные и праздничные дни × 2.0 (ст. 153)
  • Больничные по 255-ФЗ:
    стаж < 5 лет → 60%, 5–8 лет → 80%, ≥ 8 лет → 100%
    Стаж считается автоматически по дате найма
  • Отпускные по ТК РФ ст. 139:
    оклад / 29.3 × количество дней отпуска

  Все коэффициенты (1.5, 2.0, порог сверхурочных)
  настраиваются администратором без изменения кода —
  через страницу «Настройки».

  Профиль сотрудника показывает детальную разбивку
  зарплаты с навигацией по месяцам.


── ОТЧЁТЫ И АНАЛИТИКА (30 сек) ──────────────────────────

  • Сводный отчёт по подразделениям — часы, сверхурочные,
    ФОТ, явки/больничные/прогулы за любой месяц
  • Экспорт в Excel (xlsx), CSV и текстовый формат
  • Выявление аномалий: автоматическая пометка
    сотрудников с > 30% прогулов или > 10 больничных
  • Профиль каждого сотрудника — история по месяцам,
    статистика за всё время, предупреждения о прогулах


── УПРАВЛЕНИЕ СОТРУДНИКАМИ (15 сек) ─────────────────────

  • CRUD с валидацией, поиск и фильтрация
  • Мягкое удаление (деактивация) — история не теряется
  • Импорт из CSV — можно загрузить штат одним файлом
  • Дата найма → автоматический расчёт стажа


── ПОЧЕМУ МЫ ЛУЧШЕ АЛЬТЕРНАТИВ (30 сек) ────────────────

Есть три типичных пути, которые выбирают предприятия:

  1. Excel — нет разграничения доступа, нет расчёта,
     файлы теряются, одновременная работа невозможна

  2. 1С:Зарплата и управление персоналом — мощно,
     но дорого, требует отдельного внедрения, лицензий
     и специалиста на поддержку. Избыточно для задачи
     учёта табеля

  3. Коробочные SaaS-решения — не адаптированы под
     специфику оборонного производства, данные уходят
     на сторонние серверы (что неприемлемо для ТАНТК)

  Наше решение: open source, развёртывается на
  собственных серверах предприятия, полностью
  адаптировано под реальный процесс завода,
  дорабатывается под любые требования.


── ДОРОЖНАЯ КАРТА — ЧТО МОЖНО ДОБАВИТЬ (30 сек) ────────

При реальном внедрении логичны следующие шаги:

  Краткосрочно (1–3 мес.):
  • Мобильное приложение для мастеров — отметки
    прямо со смартфона на производстве
  • Уведомления (email / Telegram) — автооповещение
    руководителя при превышении нормы прогулов
  • График отпусков — планирование и согласование

  Среднесрочно (3–6 мес.):
  • Интеграция с СКУД (система контроля доступа) —
    автоматический импорт отметок с турникетов
  • Интеграция с 1С — выгрузка готовых данных
    для расчёта зарплаты без ручного переноса
  • Электронная подпись табелей (ЭЦП)

  Долгосрочно:
  • BI-аналитика — прогнозирование фонда оплаты труда,
    анализ загрузки цехов, сезонность больничных
  • Мультитенантность — масштабирование на дочерние
    предприятия холдинга


── ИТОГ (20 сек) ────────────────────────────────────────

За время хакатона мы построили полноценную рабочую
систему, которую можно развернуть на реальном
предприятии прямо сейчас — без лицензий, без внешних
зависимостей и с полным соответствием ТК РФ.

  ✓ Быстро — табель на весь цех за минуты
  ✓ Прозрачно — каждая цифра в зарплате объяснима
  ✓ Безопасно — данные только на серверах завода
  ✓ Масштабируемо — от 50 до 10 000 сотрудников

Система готова к демонстрации. Спасибо за внимание.
